Resynchronize the state after a rule application.

It takes the function to store the new activities as an argument whose signature is store rule_id syntactic_rule_id new_activity.

As long as you don't use any algebraic variable (that include don't pick a rule randomly), you can apply several rules in row before resynchronizing. (This is what initial state and perturbations do.)
